# MedVerify — Docker demo

A self-contained two-container stack that brings up a working `/verify` endpoint in under two minutes (after first-run image build), with a Neo4j seeded by a 5K-edge subset of OptimusKG.

## Run

```bash
make demo            # build + up + smoke test
make demo-down       # stop and remove
```

## What it gives you

| Service | URL | Notes |
|---------|-----|-------|
| MedVerify API | http://localhost:8000 | Try `/docs` for the OpenAPI explorer |
| Neo4j Browser | http://localhost:7474 | `neo4j` / `medverifydemo` |

The seeded KG covers ~50 high-traffic drugs (Metformin, Lisinopril, Aspirin, Warfarin, Atorvastatin, …) and the diseases / phenotypes they treat or contraindicate. It is enough to show:

- Clean `SUPPORTED` verdict — `Metformin treats Type 2 Diabetes`
- SapBERT lay-term recovery — `Heart attack treated with aspirin` (maps to *myocardial infarction*)
- Opposite-probe `CONTRADICTED` — `Insulin is contraindicated in diabetes`

## Limitations

- LLM predicate fallback is **disabled** in the demo (no API keys baked in).
- The seeded KG is a slice of OptimusKG, not the full ~1.6M edges. Claims about drugs / diseases outside the curated set will return `NOT_FOUND`.
- GLiNER weights (~1.3 GB) download on first `/verify` call and are cached in a Docker volume so subsequent restarts are fast.
- For the production-grade KG, run `make setup` (no Docker) and connect to your own Neo4j Aura instance.

## Refreshing the demo subset

The seed file `docker/neo4j-seed/load_demo_subset.cypher` is generated by `scripts/build_demo_subset.py` against the full OptimusKG parquet files. To regenerate after a data refresh:

```bash
python tools/download_optimuskg.py     # if you don't already have data/optimuskg/
python scripts/build_demo_subset.py
git diff docker/neo4j-seed/load_demo_subset.cypher
```
